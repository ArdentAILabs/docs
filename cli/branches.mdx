### ardent branch create

Create a new isolated database branch. Automatically checks it out and returns the connection URL.

```bash
ardent branch create my-feature
```

<Accordion defaultOpen={false} title="Example output">
```
✓ Branch 'my-feature' created and checked out in 4.2s

postgresql://user:pass@my-feature-postgres.routing.tryardent.com:5432/db
```
</Accordion>

### ardent branch list

In the terminal, the **current** branch is **green**: the `*`, health dot, name, and connection URL. Other branches use default color for the name and **dim** for the URL. `●` means **active**, `○` means not active. Each URL is indented on the line below the branch name (six spaces in the terminal). There is a **blank line** between each branch.

Offline cache shows `⚠ Offline - showing cached data from …` above the list when applicable.

```bash
ardent branch list
```

<Accordion defaultOpen={false} title="Example output (colors omitted)">
```
Branches:
* ● my-feature
      postgresql://user:pass@my-feature-postgres.routing.tryardent.com:5432/db

  ○ staging
      postgresql://user:pass@staging-postgres.routing.tryardent.com:5432/db

```
</Accordion>

### ardent branch info

Show details and connection URL for a branch. Defaults to the current branch if no name is given.

```bash
ardent branch info my-feature
```

<Accordion defaultOpen={false} title="Example output">
```
Name:    my-feature
URL:     postgresql://user:pass@my-feature-postgres.routing.tryardent.com:5432/db
Status:  active
Created: 2 minutes ago
```
</Accordion>

### ardent branch switch

Switch to a different branch.

```bash
ardent branch switch staging
```

<Accordion defaultOpen={false} title="Example output">
```
✓ Switched to staging
```
</Accordion>

### ardent branch delete

Delete a branch. Branches auto-suspend after 5 minutes of inactivity, so cleanup is optional.

```bash
ardent branch delete my-feature
```

<Accordion defaultOpen={false} title="Example output">
```
✓ Branch deleted: my-feature
```
</Accordion>
